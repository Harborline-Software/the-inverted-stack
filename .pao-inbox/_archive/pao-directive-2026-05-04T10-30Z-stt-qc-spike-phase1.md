## What you are doing - Phase 1 spike execution

### Step 1 - Install STT

```bash
pip3 install faster-whisper
```

If pip3 install fails on Python 3.9.6 (faster-whisper depends on
ctranslate2; ctranslate2 wheels exist for py39 macOS, should resolve), fall
back to `pip3 install openai-whisper` and use the `whisper` CLI form. Note
in the beacon which library you ended up using.

### Step 2 - Write a small spike runner

Create `build/stt_spike.py` (gitignore not needed; build/ is tracked). Keep
it tight - this is a spike script, not a polished tool. The script should:

1. Take a chapter MP3 path + the corresponding source markdown path.
2. Run STT (whisper-large-v3 if available; whisper-base as fallback if
   wall-time is unbearable on Mac CPU - note which model in the report).
3. Capture full transcript + per-segment timestamps + per-segment confidence
   (faster-whisper returns these via `model.transcribe(..., word_timestamps=True)`).
4. Normalize both source markdown and STT transcript:
   - Strip code fences (` ``` ... ``` `) and inline code spans.
   - Strip Markdown formatting markers (`*`, `_`, `#`, `>`, `[…](…)`).
   - Strip frontmatter (everything between leading `---` markers).
   - Lowercase everything.
   - Strip punctuation (.,;:!?"'-–) but keep word boundaries.
   - Collapse whitespace.
5. Diff using `difflib.SequenceMatcher` or `difflib.ndiff` at the word level.
6. Emit a report file `build/output/stt_spike/<chapter-stem>_diff.md` with:
   - Word counts (source / transcript / diff regions)
   - Each diff region inline with surrounding context (±10 words)
   - **Manual classification column** (left blank for you to fill in during
     review) with the four categories from the plan: REAL / VARIANT / NOISE /
     FOREIGN

### Step 3 - Run on ch01 + ch15

Run the script against both chapters. Wall-time budget: ≤30 min per chapter
on Mac CPU using whisper-large-v3 at 55M MP3 size. If the first chapter
exceeds 60 min, abort and switch to whisper-medium or whisper-base.

```bash
python3 build/stt_spike.py \
  build/output/audiobook/ch01-when-saas-fights-reality.mp3 \
  chapters/part-1-thesis-and-pain/ch01-when-saas-fights-reality.md

python3 build/stt_spike.py \
  build/output/audiobook/ch15-security-architecture.mp3 \
  chapters/part-3-reference-architecture/ch15-security-architecture.md
```

### Step 4 - Manually classify diff regions

Fill in the classification column in each report. Use the rubric:

- **REAL** - TTS dropped, duplicated, or audibly mispronounced. Use the
  timestamp to spot-listen if uncertain.
- **VARIANT** - TTS pronounced correctly, STT transcribed differently
  (e.g., "Lakshmi" → "lockmee"). NOT a real error.
- **NOISE** - Punctuation / em-dash / quote / formatting artifact in the
  diff that survived normalization. Refine normalization in step 2 if a
  whole class shows up.
- **FOREIGN** - Bangla, Spanish, Hindi, Portuguese passages. STT confidence
  drops naturally on these. Flag separately.
- **STALE** - Cast-swap residue in audio (Lakshmi/Lars/Étienne/Petra/Aisha)
  that has been replaced in source text. Document but do not count against
  any category.

### Step 5 - Tally and decide

Per the plan's success criterion (Phase 0, item 4):

- **Spike PASSES** if ≥1 REAL error per chapter AND FP ratio (NOISE+VARIANT)
  : REAL is < 3:1.
- **Spike LOWER-PASSES** if ≥1 REAL per 5 chapters with FP ratio < 5:1.
- **Spike FAILS** otherwise.

Note: with only 2 chapters in this spike, the "≥1 REAL per 5 chapters"
lower bar is statistically weak - extrapolate honestly in the report.
